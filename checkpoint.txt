CRO Doctor — Checkpoint (2026-04-17)
=====================================

WORKING STATE
The full onboarding and dashboard pipeline is operational: Shopify OAuth flow
(GET /auth/install → GET /auth/callback) upserts the Store with setupStatus=SYNCING,
finds-or-creates a User by email, triggers a background product sync +
captureWindowedBeforeSnapshot for every product, then sets setupStatus=COMPLETED.
The frontend OnboardingLoading component (/app/onboarding) subscribes to the
Store table via Supabase Realtime and redirects to /dashboard on COMPLETED.
The dashboard "Recommended Next Moves" section was broken by a CORS mismatch
(ALLOWED_ORIGIN defaulted to Vite port 5173; Next.js runs on 3001) — fixed by
setting ALLOWED_ORIGIN=http://localhost:3001 in api/.env (not committed; set
manually after each clone).

MANUAL STEPS REQUIRED ON A FRESH CLONE

2. web/.env.local: fill in NEXT_PUBLIC_SUPABASE_URL and NEXT_PUBLIC_SUPABASE_ANON_KEY
3. Supabase SQL Editor: ALTER PUBLICATION supabase_realtime ADD TABLE "Store";
4. cd api && npx prisma migrate deploy && npx prisma generate
